24/*******************************************************************************
25 * TRC_CFG_APPTRACE_TRAX_BLOCKING_MODE
26 * This stream port for streaming over XTensa Trax relies on apptrace that utilize
27 * an internal TRAX RAM buffer read by OpenOCD during execution.
28 *
29 * Possible values:
30 * - APPTRACE_TRAX_MODE_BLOCK_IF_FIFO_FULL
31 * - APPTRACE_TRAX_MODE_NO_BLOCK_SKIP (default)
32 *
33 * Using APPTRACE_TRAX_MODE_BLOCK_IF_FIFO_FULL ensure that you get a
34 * complete and valid trace. This may however cause blocking if your streaming
35 * interface isn't fast enough, which may disturb the real-time behavior.
36 *
37 * We therefore recommend APPTRACE_TRAX_MODE_NO_BLOCK_SKIP. In this mode,
38 * Tracealyzer will report lost events if the transfer is not
39 * fast enough. If there are problems, try increasing the apptrace buffer
40 * in sdkconfig.
41 ******************************************************************************/
